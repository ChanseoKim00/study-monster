# 스터디 그룹 출석/경고/벌금 자동화 앱 — 기획 스펙

이 문서는 Claude Code(Opus)가 바로 구현을 시작할 수 있도록 작성된 기획서입니다. 기획자(나)는 의도와 규칙을 정의했고, 구현 방식(코드)은 Opus가 자유롭게 설계해도 됩니다. 단, "배경 및 의도" 섹션의 맥락을 이해하고, "확정된 규칙"은 임의로 바꾸지 말고, "구현자가 결정할 부분"은 합리적으로 판단해서 진행해주세요.

---

## 1\. 배경 및 의도

친구들끼리 매일 밤 페이스톡을 켜고 같이 공부하는 모임을 운영 중. 문제는:

1. **학습 분위기를 망치는 친구를 강제 퇴장시키기가 사회적으로 부담스러움** → 사람이 직접 내쫓는 대신, "경고가 누적되면 시스템이 자동으로 퇴장시킨다"는 규칙으로 책임을 시스템에 위임하고 싶음. 친구 사이 감정 소모를 줄이는 게 핵심 목적.  
2. **벌금을 강제로 걷기가 힘듦** → 사람이 독촉하는 대신 시스템이 자동으로 계산/안내까지 해줘서, "시스템이 그렇게 정했다"는 명분을 만들어주고 싶음.

즉 이 앱의 핵심 가치는 **"친구 사이의 불편한 강제력 행사를 시스템 규칙으로 대체하는 것"**. 기능을 만들 때 이 목적을 항상 우선순위에 둘 것 — 예를 들어 판정이 모호하면 "봐주기"보다는 "규칙대로 명확히 처리"하는 쪽을 택해야 애초에 시스템에 위임하는 의미가 있음.

## 2\. 확정된 운영 규칙 (절대 규칙 — 변경 금지)

### 2.1 기본 스터디 시간

- 매일 **22:00 \~ 23:00** 참여  
- 23:00이 되면 **당일 학습 내용을 기재**하고 마무리

### 2.2 지각/결석 판정

| 상황 | 처리 |
| :---- | :---- |
| 무단 지각 | 경고 0.5회 |
| 무단 결석 | 경고 1회 |
| 정당한 사유로 지각/결석 | 경고 없음 |
| 애매한 개인사정으로 결석 (예: "개인사정") | **무단으로 처리** (경고 1회) |
| 야근, 일이 늦게 끝남 | **정당한 사유로 자동 인정** (경고 없음) |
| 운동 등 다른 자기개발 시간과 겹침 | **정당한 사유로 자동 인정** (경고 없음) |
| 학습 분위기 저해 (카메라 들고 돌아다님, 공부와 무관한 행동, 심한 노출, 웃고 떠드는 모습 포착 등) | 경고 0.5회 (다른 판정과 **별도로 추가** 가산) |

### 2.3 경고 초기화

- **매주 월요일**에 경고 초기화 (과거 데이터를 삭제하라는 뜻이 아니라, 주(week) 단위로 집계 기준이 새로 시작된다는 뜻 — 이력은 보존하고 "이번 주 누적치"만 0으로 보는 방식을 권장)

### 2.4 벌금

- **경고 2회 이상 누적 시 벌금 5,000원**  
- 예외: 술약속 등 **개인사정으로 주 2회 이내** 참여 안 하는 것은 벌금 부과하지 않음  
  - → 이 예외는 "애매한 개인사정 결석"이 주 2회 이내이고, **다른 사유(무단지각, 분위기저해 등)로 인한 경고가 섞이지 않았을 때만** 적용. 개인사정 결석 1\~2회 \+ 무단지각 1회가 겹쳐서 2점을 넘기면 정상적으로 벌금 부과 (즉 "개인사정 2회까지는 눈감아준다"는 거지 "경고 2점 넘었는데 무조건 면제"가 아님). 이 해석이 맞는지는 기획자(나)에게 한 번 더 확인 필요 — 모호하면 일단 이 해석으로 구현.

### 2.5 자동 퇴장 (이번 작업의 핵심 신규 요구사항)

기존엔 없던 규칙. 아래 정책으로 신규 설계 필요:

- 경고/벌금이 누적되는 친구를 **사람이 판단해서 내쫓지 않고, 시스템이 정해진 기준에 따라 자동으로 스터디방에서 제외**해야 함  
- 구체적 임계치(예: "연속 N주 벌금 대상이면 자동 퇴장")는 기획자가 아직 확정하지 않음 → **구현자가 합리적인 기본값을 제안하고, 관리자가 나중에 조정 가능하도록 설계** (예: 연속 3주 벌금 대상 시 경고 안내, 연속 4주 시 자동 퇴장 — 단 이건 제안일 뿐, 코드에 하드코딩하지 말고 설정값으로 분리)

### 2.6 벌금 정산 (이번 작업의 핵심 신규 요구사항)

- 친구들 사이라 벌금을 강제로 걷기 힘듦 → **돈을 누군가에게 몰아주는 방식이 아니라, 모인 벌금을 매주 혹은 매달 정산해서 그 기간 학습방 참가자 수로 n분의 1 해서 전원에게 환급**하는 방식  
- 즉 벌금은 "처벌"이지만 동시에 "참여자 전체의 공동 보상 풀"로 기능함 (개근한 사람도 결국 n분의 1을 받아가니, 지각/결석한 사람이 모두에게 보상을 나눠주는 구조)  
- **실제 계좌 송금까지는 이번 범위에 포함하지 않음.** "이번 정산 기간 모인 벌금 총액 / 참가자 수 \= 1인당 환급액"을 계산해서 보여주는 것까지가 범위. (사람이 보고 카톡으로 보내거나 정산)

## 3\. 출결 판정 로직 (구현 디테일)

### 3.1 출석 인정 조건

시스템이 자동으로 입장 시각/종료 시각을 기록해야 함 (화상통화를 앱 자체에 내장하기로 결정 — 3번 섹션 "기술 결정사항" 참고). 출석 인정 조건:

- **시작 후 5분 이내 입장** AND **전체 학습 시간의 95% 이상 체류**  
- 위 두 숫자(5분, 95%)는 **관리자가 조정할 수 있는 설정값**이어야 함 (하드코딩 금지)

판정 매트릭스 (제안 — 기획자 확인 필요):

| 입장 시각 | 체류율 | 판정 |
| :---- | :---- | :---- |
| 5분 이내 | 95% 이상 | 정상 출석 |
| 5분 이내 | 95% 미만 (조퇴/이탈) | 결석 처리 (시간 맞춰 들어왔어도 충분히 머물지 않았으므로) |
| 5분 초과 | 95% 이상 | 지각 |
| 5분 초과 | 95% 미만 | 결석 처리 (둘 다 미달이므로 더 무거운 결석으로 처리) |
| 아예 미입장 | \- | 결석 |

→ **이 매트릭스는 기획자가 확정한 게 아니라 합리적으로 추론한 것.** 애매하면 이 표를 기본값으로 구현하되, 코드 주석에 "이 판정 기준은 추정치이며 운영하면서 조정 가능성 있음"이라고 남겨둘 것.

### 3.2 경고 가중치

모두 관리자 설정값으로 분리 (기본값):

- 무단 지각: 0.5점  
- 무단 결석: 1.0점  
- 애매한 개인사정 결석: 1.0점 (무단 결석과 동일 취급)  
- 학습 분위기 저해: 0.5점 (다른 판정에 **가산**, 중복 가능 — 예: 지각 0.5 \+ 분위기저해 0.5 \= 총 1.0점)

### 3.3 정당한 사유 카테고리

- 야근/업무 → 자동 정당 (경고 없음)  
- 운동 등 자기개발 일정 → 자동 정당 (경고 없음)  
- 애매한 개인사정 → 무단 처리 (경고 있음, 단 주 2회까지 벌금 면제)  
- 기타 → 관리자 판단 필요 (수동 승인 플로우 필요)

이 사유 입력은 **멤버가 직접 신고**하는 방식으로 설계 (예: 지각/결석 예상 시 미리 또는 사후에 "사유" 선택). 사유 미입력 시 기본은 무단 처리.

### 3.4 학습 분위기 저해 판정

**AI/시스템이 자동으로 감지하는 건 이번 범위 밖** (웹캠 영상 분석은 기술적으로 너무 무겁고 사생활 문제도 있음). 대신:

- 멤버가 화상방 화면에서 "분위기 저해 신고" 버튼으로 신고  
- 또는 관리자가 수동으로 특정 멤버/세션에 플래그 지정  
- 신고가 들어오면 0.5점 경고 자동 부여 (반복 신고 시 누적되는지, 1세션당 1회만 적용되는지는 구현자가 합리적으로 결정 — 제안: 1세션당 최대 1회만 적용해서 어뷰징 방지)

## 4\. 기술 결정사항 (이미 확정함)

- **화상통화**: 페이스톡 대신 **앱 자체에 화상 기능을 내장**하기로 결정. 외부 화상 SDK(Daily.co, Agora, LiveKit 등 중 구현자가 선택)를 붙여서, 입장/퇴장 이벤트(webhook)로 자동 출석 추적  
  - 이걸 선택한 이유: 외부 메신저(페이스톡)는 우리 시스템이 입퇴장을 알 방법이 없어서 자동화가 불가능했음. 직접 화상 기능을 만드는 건 시그널링 서버 등 너무 큰 작업이라, SDK를 붙이는 게 합리적 선택.  
- **배포 환경**: 기획자가 **Railway 유료 플랜**을 이미 사용 중 → 백엔드(Node/Next.js 등) \+ PostgreSQL DB를 Railway에 올릴 수 있음. 구현자는 이 환경에 맞춰 스택을 선택하면 됨.  
- **실제 송금 자동화**: 범위 밖 (PG 연동 안 함). 계산까지만.  
- **분위기 저해 자동 감지(영상 분석)**: 범위 밖. 신고 기반으로 처리.

## 5\. 핵심 기능 목록 (우선순위 순)

1. 화상방 입장 시 입/퇴장 시간 자동 기록  
2. 출결 자동 판정 (5분/95% 기준, 관리자 조정 가능)  
3. 경고 자동 부여 \+ 주간 집계 (월요일 기준)  
4. 사유 신고 (정당한 사유 / 애매한 개인사정 선택 UI)  
5. 분위기 저해 신고 버튼  
6. 주간 벌금 자동 판정 (경고 2점 이상, 단 개인사정 2회 이내 예외 처리)  
7. **자동 퇴장 시스템** (신규 — 연속 벌금 누적 시 시스템이 판단해서 멤버 비활성화)  
8. **벌금 정산/n분의1 환급 계산** (신규 — 주간 또는 월간, 관리자가 선택)  
9. 23:00 마감 시 학습 내용 기재  
10. 관리자 설정 화면 (위 모든 숫자 기준값 조정)

## 6\. 기획자가 아직 확정하지 못한 부분 (Opus가 구현 시 합리적 기본값 제안 \+ 설정값으로 분리해둘 것)

- 자동 퇴장 임계치 (연속 몇 주 벌금이면 퇴장인지)  
- "개인사정 2회 면제" 규칙이 다른 경고와 섞였을 때 정확히 어떻게 적용되는지 (3.2 해석 참고)  
- 분위기 저해 신고가 한 세션에 여러 번 들어왔을 때 중복 가산 여부  
- 벌금 정산 주기를 주간/월간 중 무엇으로 할지 (관리자가 선택하게 하는 게 안전)  
- "기타" 사유 신고에 대한 관리자 수동 승인 플로우의 구체적 UX

이 부분들은 막히지 말고, 가장 합리적인 기본값으로 구현 후 "추정해서 구현했고 운영하면서 조정 필요"라고 표시해주면 됨.

## 7\. 핵심 로직 검증 (참고용 — 이미 시제품에서 테스트 완료)

아래는 이전에 만들어본 시제품에서 실제로 통과한 테스트 케이스. 새로 구현할 때 동일한 케이스들을 검증에 활용하면 좋음:

- 정시 입장(5분 이내) \+ 95% 이상 체류 → 정상 출석  
- 10분 지각 \+ 95% 이상 체류 → 지각, 경고 0.5점  
- 미입장 → 결석, 경고 1.0점  
- 야근 사유 신고 → 정당한 사유, 경고 0점  
- 애매한 개인사정 신고 → 무단 처리, 경고 1.0점  
- 정상 출석 \+ 분위기 저해 신고 → 경고 0.5점 (출석이어도 가산됨)  
- 개인사정 결석만 주 2회 → 벌금 면제  
- 무단 지각/결석으로 주간 경고 2점 누적 → 벌금 부과  
- 벌금 15,000원 / 참가자 4명 → 1인당 환급 3,750원  
- 연속 4주 벌금 대상 → 자동 퇴장

## 8\. Opus에게 전달하는 한 줄 요약

"친구들끼리 매일 22\~23시 화상 스터디 출석을 자동 추적하고, 지각/결석/분위기저해를 규칙대로 경고·벌금으로 환산하고, 벌금을 모아 참가자 전원에게 n분의 1 환급하며, 경고가 심한 친구는 사람이 아니라 시스템이 자동으로 내쫓는 — '친구 사이 불편한 강제력을 시스템에 위임하는' 스터디 관리 앱을 Next.js \+ PostgreSQL \+ 화상 SDK(Daily.co 등) \+ Railway 배포로 만들어주세요. 판정 기준은 전부 관리자가 조정 가능한 설정값으로 분리해주세요."
